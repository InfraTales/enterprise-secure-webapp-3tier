# Contributing to InfraTales

Thank you for your interest in contributing! We welcome improvements, bug fixes, and new features.

## Workflow
1.  **Fork** the repository.
2.  Create a **feature branch** (`git checkout -b feature/amazing-feature`).
3.  **Commit** your changes (`git commit -m 'Add amazing feature'`).
4.  **Push** to the branch (`git push origin feature/amazing-feature`).
5.  Open a **Pull Request**.

## Guidelines
-   **Tests:** Ensure all unit and integration tests pass.
-   **Docs:** Update `README.md` and `docs/` if you change architecture or configuration.
-   **Diagrams:** Update Mermaid diagrams if the flow changes.
-   **Linting:** Ensure code is formatted according to the project standards.
